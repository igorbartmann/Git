Estudando Git e GitHub

Igor Bartmann

Porque utilizar o Git:

O Git é uma ferramenta que possibilita criar repositórios de código-fonte e arquivos, visando ter um maior controle das versões.

Através do Git é possível guardar todo o histórico de alterações do projeto, assim como voltarmos à versões anteriores. Devido a esse versionamento, o Git se tornou muito útil para trabalhos em equipe, quando vários programadores trabalham, juntos, sobre um mesmo projeto. Dessa forma é possível que diferentes programadores trabalhem ao mesmo tempo em um único projeto, cada um criando diferentes versões que, no final, serão unidas em uma única.

Como Instalar o Git:

Para realizar o download do Git, acesse a página oficial da ferramenta: Clique aqui para ir para a página oficial do Git!

Na página oficial do Git, terá um espaço destinado apenas ao download. Faça o download para o seu Sistema Operacional e em seguida realize a Instalação. Para começar a utilizar o Git, você pode manter todas as configurações que já vem previamente estabelecidas na hora de realizar a instalação.

Conceitos Básicos do Git:

Commit

Commit é como se fosse um "checkpoint" de modificação, ou seja, é um ponto para indicar que naquele momento houve uma modificação no projeto.

Normalmente devemos gerar um commit sempre que a nossa base do código tiver uma mudança que consideramos importante. Mas por Boa-Prática, nunca devemos commitar um código que não funciona

Branch

Branch são ramos utilizados para separar o desenvolvimento de diferentes funcionalidades dentro de um projeto.

O branch master é o branch “padrão” de um repositório.

É interessante separar o desenvolvimento em branches diferentes, para que as mudanças no código de uma, não influencie no funcionamento de outra.

Primeiros Passos com o Git:

Junto com a instalação do Git, será instalada uma ferramenta muita importante para a sua utilização: O Git Bash.
É através desta ferramenta que nos comunicaremos com o Git, utilizando os comandos pré-estabelecidos.

Antes de mais nada, crie um diretório dentro do seu computador e nele, adicione um arquivo qualquer(.html, ou até mesmo .txt).

Agora, abra o Git Bash e navegue até este diretório que você acabou de criar. Para isso, utilize os comandos:

  • cd <folderName> - Navegar para o diretório informado.
  • cd .. - Para voltar para o diretório anterior.

Quando você já estiver dentro do diretório que acabou de criar, digite os seguintes códigos:

  • git init - Faz do diretório atual, um repositório Git;
  • A primeira configuração a fazer, é indicar "Quem você é" ao Git:
    • git config user.name "SeuNome" - Configura o seu nome;
    • git config user.email "SeuEmail" - Configura o seu email;
  • git status - Exibe o estado dos arquivos encontrados;
  • git add <fileName> - Adiciona o arquivo informado ao monitoramento do Git;
  • git commit -m "Mensagem" - Commita o estado atual do branch, dando a este commit a mansagem informada;
  • git log - Exibe os commits realizados, como uma espécie de histórico;

Note que sempre que você realizar uma modificação no arquivo do repositório (diretório criado por você)e escrever o código "git status", verá que a versão mais atual do arquivo não estará adicionada ao git, para isso você terá que adicionar a nova versão do arquivo ao Git (git add) e realizar um novo commit (git commit -m "Mensagem").

Criando Repositórios com o GitHub:

Acesse o site oficial do GitHub - Clicando aqui! - , crie uma conta e faça login.

Após isso, clique em "Novo", para adicionar um Novo Repositório. Em seguida configure o repositório, escolhendo, entre outros itens, o nome e a visibilidade do seu repositório.

Ao criar o repositório, aparecerá uma página com vários modelos de códigos, mas o mais importante agora, é você copiar a URL/Link da página do seu repositório.

Feito isso, volte ao Git Bash, navegue até o diretório criado por você e digite o seguinte código:

  • git remote add servidor <UrlDoSeuRepositórioGitHub> - Com isso você adicionará um novo repositório remoto (chamado "servidor", poderia ser outro nome qualquer) para o seu repositório local.
  • git remote - Exibe uma lista dos repositórios remotos encontrados para o seu repositporio local (verifique se aparecerá o nome recem adicionado - "servidor").
  • git push servidor master - Com isso todos os dados do seu branch (chamado "master") serão enviados ("push") para o seu repositório remoto (chamado "servidor").

A partir dai, sempre que você modificar algo no seu projeto atual, adicione a nova versão ao git (git add), faça um novo commit (git commit -m "Mensagem") e envie os dados atualizados para o repositório remoto - GitHub - (git push servidor master).

Mas... e se você estiver em outro computador, como trazer os dados do repositório remoto - GitHub - para este outro computador?

  1. a) Crie um novo diretório dentro deste computador.
  2. b) Através do Git Bash (se o computador não possuir, precisará ser realizado o download), navegue até o diretório atual.
  3. c) Quando chegar dentro do navegatório recem criado, digite o código git init (para tornar este diretório um repositório git), seguido pelos códigos de configurar "Quem você é" (git config user.nome "Seu Nome" e git config user.email "Seu Email").
  4. d) Após isso, adicione um repositório remoto - GitHub - a este diretório (git remote add servidor <UrlDoSeuRepositórioGitHub>).
  5. e) Para trazer os dados do repositório remoto - GitHub - para dentro do computador, utilize o código git pull servidor master. Com isso todos os dados do diretório remoto ("servidor") serão puxados ("pull") para dentro do branch atual ("master").

Lembre-se, isto é o básico que você precisa saber para conseguir trabalhar com Git e GitHub.
